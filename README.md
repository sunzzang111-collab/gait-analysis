# 보행 분석 (참고용)

브라우저에서 바로 동작하는 카메라 기반 보행 분석 도구입니다. 실시간 카메라 또는
업로드한 영상에서 사람의 관절 위치를 추정하고, 관절 각도 · 좌우 대칭성 ·
케이던스(보행 주기) 등을 계산해 화면에 보여줍니다.

> **참고용 소프트웨어입니다.** 의료기기가 아니며 진단 목적으로 사용할 수 없습니다.
> 모든 수치는 의료진의 임상적 판단을 보조하는 참고 자료로만 사용하세요.

## 주요 기능

- **실시간 카메라 분석**: 진료실 노트북/태블릿 카메라로 환자가 걷는 모습을 바로 분석
- **영상 업로드 분석**: 미리 촬영한 보행 영상을 업로드해 분석
- **관절 각도**: 무릎 · 골반 · 발목 각도를 좌/우 실시간 표시 + 시간별 그래프
- **좌우 대칭성**: 관절 가동범위(ROM), 보행주기, 상대 보폭의 좌우 대칭성 차이(%)
- **케이던스**: 분당 걸음 수

모든 포즈 추정은 **브라우저 안에서만** 실행됩니다(Google MediaPipe Pose
Landmarker, WASM). 영상이 서버로 전송되지 않으므로 환자 영상이 외부에 저장되지
않습니다.

## 촬영 가이드

- 환자의 **옆모습(측면)** 이 프레임에 온전히 들어오도록 촬영하세요.
- 카메라는 허리 높이 정도로 고정하고, 환자가 5~10걸음 정도 걷는 모습을 촬영합니다.
- 전면 카메라(셀카 모드)처럼 화면이 좌우 반전되어 보이는 경우, 화면 상단의
  "카메라가 좌우 반전됨" 체크박스를 켜면 좌/우 라벨이 보정됩니다.
- 절대 거리(cm 단위 보폭 등)는 카메라 보정 없이는 제공하지 않습니다. 보폭은
  환자의 다리 길이(엉덩이-발목 거리)를 기준으로 한 **상대값**입니다.

## 지표 산출 방식 (참고)

- **관절 각도**: 인접한 세 관절 좌표로 이루는 각도(예: 무릎 = 골반-무릎-발목).
- **케이던스/보행주기**: 발목의 화면상 수직 위치에서 지면에 닿는 시점(국소 최고점)을
  검출해 걸음 시점으로 사용합니다. 카메라가 보행 방향과 수직(측면)일 때 가장
  안정적입니다.
- **대칭성 차이(%)**: `|좌-우| ÷ 평균 × 100`. 0%에 가까울수록 좌우가 대칭적입니다.

이 방식은 단일 카메라 기반의 근사치이며, 3D 모션캡처나 압력판 기반 보행분석
장비와 같은 정밀도를 갖지 않습니다.

## 개발

```bash
npm install
npm run dev      # 개발 서버
npm run build    # 프로덕션 빌드
npm run preview  # 빌드 결과 미리보기
```

## 기술 스택

- React + TypeScript + Vite
- [@mediapipe/tasks-vision](https://ai.google.dev/edge/mediapipe/solutions/vision/pose_landmarker) (Pose Landmarker, 브라우저 내 추론)
- Recharts (지표 그래프)

서버/백엔드 없이 정적 파일로 배포 가능한 SPA입니다(Vercel, Netlify, GitHub Pages 등).
